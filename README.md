# PlayerTicker

A fantasy football **market terminal** — it tracks player value like a stock ticker so managers can
spot risers, fallers, buy-low windows, and overheated assets before their league reacts.

This repository is the **MVP / P0 build** described in [`DESIGN.md`](./DESIGN.md): a polished,
mobile-first, dark-mode SPA running entirely on a clearly-labeled **Demo Market** — a deterministic,
seeded mock-data engine that simulates daily value movement for ~140 fantasy-relevant players. No
login, no backend, no live data.

> **Demo Market — simulated player values for product preview. Not current player information.**
> Every value on the site is generated by the internal market engine from authored inputs.

## Run it locally

```bash
npm install
npm run dev        # http://localhost:5173
```

Other scripts:

```bash
npm run build      # type-check + production build to dist/
npm run preview    # serve the production build
npm run test       # engine + determinism unit tests (Vitest)
npm run typecheck  # tsc, no emit
```

## What's in the MVP (all P0)

- **Landing**, **Market Dashboard** ("what moved today"), **Player Market Board** (sortable/filterable
  with URL-synced state), **Player Stock Card**, **Watchlist** (localStorage, value-since-added),
  **Fantasy Portfolio** (manual, local, beta), **Methodology**, **Legal**.
- Full **market system**: market price, movement windows, mispricing, volatility, six-factor risk,
  buy/hold/sell signals with confidence + explanations, asset classes, tags, catalysts, and a
  template-generated market thesis.
- A **deterministic daily tick** — prices are a pure function of `(playerId seed, date)`, so every
  visitor sees the same values on a given day and *today differs from yesterday*.
- **Format ribbon** with six precomputed combinations (default `Dynasty · Superflex · Half-PPR`).
- A structural **honesty layer** (demo banner, freshness badges, confidence pills) wired to data
  fields, not hardcoded copy.

## Architecture

The one rule: **the UI reads only through `MarketDataService`.** Swapping the mock service for a future
live service requires zero UI changes.

```
src/
├── config/market.ts        # every weight, threshold, curve, rule (§12). The Methodology page
│                           # renders from this same file, so docs can't drift from behavior.
├── config/taxonomy.ts      # asset-class + tag definitions
├── types/market.ts         # data models (the mock<->live contract)
├── services/
│   ├── marketEngine/       # pure, config-driven functions: fundamental value, market value,
│   │                       # mispricing, volatility, risk, signals, tags, confidence (+ tests)
│   ├── marketData/         # MarketDataService interface + MockMarketDataService + dataset builder
│   ├── thesis/             # ThesisGenerator interface + deterministic template generator
│   └── storage/            # versioned localStorage
├── data/pool.ts            # authored ~140-player pool (engine INPUTS, not outputs)
├── lib/                    # prng (seeded), formatting, watchlist math, ui helpers
├── components/             # chrome/ · market/ · states/ · ui/
├── pages/                  # one file per route
└── store/                  # Zustand: format, watchlist, portfolio
```

The market engine is pure and shared: the same functions build the mock history, run the runtime tick,
and would run a future live recalculation — one formula, no drift.

## Deliberately out of scope (P1/P2)

Accounts/auth, real stats and APIs, league import, live AI/LLM calls, trade analyzer, receipts, price
alerts, admin tooling, compare, and share-image export are **not** in the MVP. Where relevant they ship
as clearly-labeled "coming soon" placeholders.

## Data & legal posture

- No scraping of any proprietary fantasy site. All values come from the internal formula over authored
  mock inputs.
- No NFL logos, marks, or licensed headshots. Player names and statistics are used factually; avatars
  are generated initials on custom, team-color-*inspired* gradients.
- Prices are fictional fantasy value indexes — not money, securities, or anything to bet on. See
  `/methodology` and `/legal`.

Stack: React 18 · Vite · TypeScript · Tailwind · React Router · Zustand · Recharts (lazy, detail chart
only). Static SPA — deployable to any static host.
